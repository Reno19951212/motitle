## Problem

After v5-A4 landed, the 賽馬娛樂新聞 video (`e496511682fb`, pipeline `ec2d55ba-...`)
still shows two classes of broken segments in the Video Preview overlay:

### 1. Primary ASR cascade hallucination clusters (zero-duration repeats)
- Segments `[52][53][54]`: same text `"所以惹來了一眾網友風暴,"` repeated 3×
- Segments `[64][65][66]`: same text `"擊敗大熱門HIGHLAND BLINK,"` 3× with
  zero/negative duration (start=205.04, end=204.96)

These cluster in the user's overlay as visually-stuck lines that flicker
into the same content multiple times.

### 2. Tail-of-clip English orphan
- Segment `[89]`: single English word `"vowels"` at 264.6s, 3.12s after the
  previous Chinese segment ends. Classic Whisper training-data leak at the
  audio tail.

Empirically — read directly from `backend/data/registry.json` — these
artifacts are present in `asr_primary.segments`, and the v5 verifier passes
them through unchanged (82/90 segments are byte-identical primary). Refiner
polishes them. They land in `translations[].by_lang[zh].text` and surface
in the Video Preview overlay.

**Root cause**: Whisper occasionally "loops" on a phrase when audio energy
is low/uncertain — it emits the same phrase multiple times with collapsed
timecodes (zero/negative duration). Tail-of-clip Whisper hallucination is
a known training-corpus leak where the decoder generates fluent-but-fake
content after the audio ends.

The v5 Verifier was *designed* to catch these via its dual-ASR LLM-as-judge
flow, but in practice the qwen3-asr secondary produces 13 mega-chunks vs
mlx-whisper's 90 short segments — granularity mismatch makes per-segment
alignment unreliable, so verifier silently defaults to "keep primary"
91% of the time. Cascade clusters slip through.

## Goals

Fix at the *source* — clean these artifacts out of `asr_primary` output
*before* they propagate to verifier / refiner / persistence. Two small
pure-function filters added to `backend/asr/segment_utils.py` (alongside
existing `split_segments` + `merge_short_segments`).

## Non-Goals

- Verifier alignment fix (still in Phase 5+ scope per v5-A4 docs).
- Backfilling existing registry entries — new runs only.
- Touching v4 ASR path — applies only to v5 `ASRPrimaryStage`.

## Architecture

```
audio
  ↓
TranscribeEngine.transcribe(audio_path)
  ↓ [raw segments — may contain cascade clusters + tail orphans]
ASRPrimaryStage.transform()
  ↓ [normalize shape]
  ↓ dedupe_cascade_repeats(segments)      ← NEW
  ↓ filter_tail_english_orphan(segments)  ← NEW
  ↓ [clean segments]
→ downstream stages (secondary / verifier / refiner / persistence)
```

Both filters are **pure functions** (immutable input, return new list)
matching the existing `segment_utils.py` style.

### Filter 1: `dedupe_cascade_repeats`

```python
def dedupe_cascade_repeats(
    segments: List[dict],
    *,
    min_duration_sec: float = 0.1,
) -> List[dict]:
    """Drop consecutive segments that repeat the previous text AND have
    near-zero or negative duration. Whisper occasionally loops on a phrase
    when audio is uncertain, emitting 2-N copies with collapsed timecodes.

    A segment is dropped when ALL of:
      - text (stripped) equals the previous KEPT segment's text (stripped)
      - duration (end - start) < min_duration_sec

    Long-duration legitimate repeats (e.g. a speaker actually saying the
    same phrase twice with normal timing) are preserved.
    """
```

Edge cases:
- Empty list → returns empty list
- Single segment → returns it unchanged
- Negative duration (start > end) → satisfies `< min_duration_sec` and is
  dropped if text repeats
- Non-repeating segments → all kept regardless of duration
- 3+ consecutive repeats → only the FIRST is kept

### Filter 2: `filter_tail_english_orphan`

```python
def filter_tail_english_orphan(
    segments: List[dict],
    *,
    max_word_chars: int = 10,
    min_gap_sec: float = 2.0,
) -> List[dict]:
    """Drop the trailing segment when it is a short pure-ASCII single word
    AND is separated from the previous segment by a gap > min_gap_sec.

    Targets Whisper's classic tail-of-clip training-data hallucination:
    'vowels', 'thanks', 'subscribe', etc. after audio has actually ended.

    Conservative — only drops if BOTH conditions hold so we don't kill
    legitimate short English brand names mid-segment list.
    """
```

A segment is the "tail orphan" when ALL of:
- It is the LAST segment in the list
- Its text matches `^[A-Za-z]+$` (pure ASCII letters, no digits / spaces / punct)
- `len(text) <= max_word_chars`
- `segments[-1].start - segments[-2].end > min_gap_sec` (gap from previous)

Edge cases:
- Empty list / single segment → returned unchanged (nothing to filter)
- Last segment is Chinese / mixed / punctuated → kept
- Last segment is English but no gap → kept (might be part of speech)
- Multi-word English tail (`"thanks for watching"`) → kept (not single word)

### Integration point: `ASRPrimaryStage.transform()`

[backend/stages/v5/asr_primary_stage.py](backend/stages/v5/asr_primary_stage.py)
currently:
```python

    engine = create_transcribe_engine(self._profile)
    language = self._profile.get("language", "auto")
    segments = engine.transcribe(self._audio_path, language=language)
```

After fix:
```python
def transform(self, segments_in, context):
    engine = create_transcribe_engine(self._profile)
    language = self._profile.get("language", "auto")
    segments = engine.transcribe(self._audio_path, language=language)
    normalized = [
        {"start": float(s["start"]), "end": float(s["end"]),
         "text": s.get("text", "").strip()}
        for s in segments
    ]
    # v5-A4.1: scrub cascade hallucination clusters + tail English orphans
    # at the source so downstream verifier/refiner/persistence see clean
    # input. Both filters are pure functions; original list unchanged.
    deduped = dedupe_cascade_repeats(normalized)
    return filter_tail_english_orphan(deduped)
```

Filters apply unconditionally — they are conservative enough that they
never false-fire on healthy ASR output (cascade dedup requires near-zero
duration; tail orphan requires gap > 2s + pure ASCII).

`dedupe_cascade_repeats`:
- Empty list → empty
- Single segment → kept
- 3 consecutive zero-duration repeats → 1 kept
- 3 consecutive normal-duration repeats → 3 kept (legitimate speech)
- Mixed: non-repeating segs interleaved → kept

- Empty / single → unchanged
- Tail = `"vowels"` (5 ASCII, gap 3s) → dropped

- Tail = `"vowel sounds"` (multi-word, gap 3s) → kept

- Tail = Chinese `"完"` (non-ASCII) → kept
- Immutability

## Risk

- **False positive on dedup**: a speaker legitimately saying the same
  phrase twice in quick succession (< 0.1s gap) would be dropped. The
  duration threshold makes this unlikely — natural speech repeats have
  real duration (≥0.3s). Acceptable.
- **False positive on tail orphan**: a final English brand name with a
  natural pause before it (`"...and the winner is. Pegasus."` if Pegasus
  ends up as the last seg) could be dropped. The 2s gap threshold means
  this only fires when there's an audible silence break before the
  trailing English word — which is exactly the hallucination signature.
- **Behavior change on v5 only** — v4 ASR path uses different stage
  classes and is unaffected.
