# Line-Budget v3.9 — A3v3 Ensemble Design Spec

**Date:** 2026-05-02
**Status:** Approved by user; ready for writing-plans phase
**Validation evidence:** [2026-05-02-line-budget-validation-tracker.md](2026-05-02-line-budget-validation-tracker.md) (45 rounds)
**Branch target:** `feat/line-wrap-v3.9-a3-ensemble` (new)

## Goal

實現繁體中文字幕分句兼顧 **Netflix TC ≤16字/行 + max 2 lines + bottom-heavy** 同 **CityU HK ≤14字/行 single-line preferred**，並 **同時保留人名/地名/術語**。

## Approach

**A3v3 Ensemble** — 三路並行翻譯（K0 baseline + K2 brevity + K4 brevity-rewrite），per-segment 揀 entity recall 最高者；之後用 lock-aware hybrid wrap（soft 14 / hard 16 / max 2 lines / bottom-heavy）。

## Tech Stack

- Python 3.9+, Flask, Flask-SocketIO（既有）
- OpenRouter `qwen/Qwen3.5-35B-A3B`（既有）
- mlx-whisper medium（ASR，既有）
- 純 stdlib + Playwright（無新 deps）

## Architecture

```
EN segments (from ASR)
        ↓
  ┌─────────┬─────────┬─────────────┐
  │ L1 K0   │ L2 K2   │ L3 K4 rewrite│  ← parallel via ThreadPoolExecutor
  │ baseline│ brevity │ K2 + per-seg │
  └────┬────┴────┬────┴──────┬──────┘
       └─────────┴───────────┘
                 ↓
        L4 A3 ensemble selector
        - find_en_entities + proxy capitalization NER
        - per-seg max(F1 recall) + CPS ≤ 9.0 gate
        - tiebreaker → K4 (shortest)
                 ↓
        L5 wrap_hybrid + V_R11 lock
        - soft_cap=14, hard_cap=16, max=2, bottom-heavy
        - 4-pass fallback chain
                 ↓
        Translated segments → renderer / preview
```

## Data Flow

### Translation Request

```
POST /api/translate {file_id}
    ↓
sentence_pipeline.translate_with_a3_ensemble(segments, glossary, profile)
    ↓
Profile flag a3_ensemble=true:
    ParallelExecutor.gather(
        L1: translate_with_baseline_prompt(segments)         → k0_segs (~10s)
        L2: translate_with_brevity_prompt(segments)          → k2_segs (~10s)
    )
    ↓
    L3: brevity_rewrite_pass(k2_segs, cap=14, entity_aware=true)
        - Per seg with len(k2_zh) > 14:
          - Extract must_keep entities from k2_zh via NAME_INDEX + proxy NER
          - LLM rewrite with explicit must_keep list
          - Validate post-rewrite: if must_keep dropped → keep k2_zh (no rewrite)
        → k4_segs (~30s sequential per long seg)
    ↓
    L4: apply_a3_ensemble(k0_segs, k2_segs, k4_segs, durations)
        For each segment i:
          en_ents = find_en_entities(en[i]) ∪ extract_proxy_entities(en[i])
          if no entities:
            winner = K4 (shortest)
          else:
            recall = {K0: count(...), K2: count(...), K4: count(...)}
            cps = {K0: chars/dur, K2: chars/dur, K4: chars/dur}
            valid = candidates with CPS ≤ 9.0
            if all valid: winner = max recall in valid (tie → K4)
            else: winner = max recall (tie → K4); flag cps-overflow
          length safety: if winner > 32c, fallback K4 + flag k4_unrescuable
        ↓
        merged_segs with .source ∈ {k0, k2, k4, k4_unrescuable}
    ↓
Store in registry: file.translations = merged_segs
Emit pipeline_timing event (G5 telemetry)
```

### Render-time

```
POST /api/render {file_id, format}
    ↓
Profile uses cityu_hybrid + a3_ensemble=true:
    if file.translations lacks K2/K4 caches (Mod 6 lazy migration):
        background-trigger translate_with_a3_ensemble
        return 202 Accepted + WS progress
    ↓
renderer.generate_ass():
    for each translation seg:
        zh = resolve_segment_text(seg, mode, order)
        locked = sentence_pipeline._build_full_lock(zh)
        result = wrap_hybrid(zh, soft=14, hard=16, max=2, tail=2, locked=locked)
        if result.flags has 'lock-violated': log warning + telemetry alert
        ass_text = '\\N'.join(result.lines)
    ↓
FFmpeg burn-in
```

### Frontend Preview

```
Page load → font-preview.js init():
    - Fetch /api/profiles/active
    - Inject @font-face for bundled fonts
    - Subscribe to socket profile_updated
    ↓
On segment change (timeupdate):
    zh = pickSubtitleText(currentSeg, profile.font.subtitle_source)
    locked = subtitleWrap.buildFullLock(zh)  // JS port of V_R11 chain
    result = subtitleWrap.wrapHybrid(zh, {soft:14, hard:16, max:2, tail:2, locked})
    ↓
fontPreview.updateText(result.lines):
    - Clear svg <text>
    - For each line: <tspan x=960 y=Y dy=...>
    - Y stacking: bottom-heavy (last line at margin_bottom, others above)
    - Total visible height = max_lines × line_height
```

## Edge Cases + Error Handling

### LLM Engine Failures

| Layer | Failure | Handling |
|---|---|---|
| L1 K0 fails | retry 3× exp backoff; if persist → degrade to K2-only ensemble; warn |
| L2 K2 fails | retry; if persist → degrade to K0-only baseline; user sees baseline output (current behavior) |
| L3 K4 rewrites fail per-seg | per-seg retry once; if fail → keep K2 zh; log; do not block other segs |
| OpenRouter rate limit (429) | pause 5s; retry; if persist → fall back to K0-only + telemetry alert |
| All 3 layers timeout | 503 to client + telemetry; user retries manually |

### Empty / Whitespace Translations

- Single layer empty → A3 selector treats recall=0; pick from remaining
- All 3 empty → fallback: zh_text = en_text + flag `empty_translation`
- Whitespace-only → strip + treat as empty

### Wrap Algorithm Pass Fallback

| Pass | Trigger | Action | Flags |
|---|---|---|---|
| 1 | Normal, BH range has unlocked break | break at best-scored unlocked position | none |
| 2 | Pass 1 found nothing in BH range | search full hard-cap range [n-hard_cap, hard_cap] | `soft_overflow` if line > soft_cap |
| 3 | Pass 2 also fully locked | search [1, n-1] for any unlocked scoring break | `soft_overflow`, possible bottom_heavy_violation |
| 4 | All [1, n-1] locked | break at min(hard_cap, n//2) — accept lock violation | `hard_cut`, **`lock-violated`** (Mod 3) |

Mod 3: Pass 4 emits `wrap_pass4_warning` event; proofread UI shows red banner; per-segment hint to manually edit.

### A3 Selector Tie-breaking

| K0 | K2 | K4 | CPS valid? | Winner |
|---|---|---|---|---|
| 2 | 2 | 2 | all | K4 (tie → shortest) |
| 1 | 2 | 1 | all | K2 |
| 2 | 1 | 0 | all, K0≤32c | K0 |
| 2 | 1 | 0 | K0=40c | K4 + flag `k4_unrescuable` |
| 0 | 0 | 0 | all | K4 |
| 2 | 1 | 1 | only K4 valid (others CPS>9) | K4 + flag `cps-overflow` (Mod 7) |

### Concurrent Job Rate Limit (Mod 8)

- Per-user semaphore: max 1 active translation
- Cross-user: global semaphore = 4 concurrent
- Duplicate request to same file → coalesce to existing job
- Queue depth telemetry monitored by G5

### Validation Gate Failures

| Gate | Failure mode | Recovery |
|---|---|---|
| G1 unit | Local dev fix |
| G2 integration | PR blocked; root cause investigation |
| G3 regression | PR blocked; possibly add new fixture for new corpus |
| G4 pre-merge | Manual review; approve-with-justification possible |
| G5 production drift (>10% F1 drop) | Auto issue + weekly review |

### Backward Compatibility

| Scenario | Behavior |
|---|---|
| Old profile without `a3_ensemble` field | default `false`; uses K0 baseline (current) |
| Old profile with broadcast preset | legacy `wrap_zh` 28/2/2 used |
| Old translation cache (no `source` field) | A3 selector treats as K0; user can re-translate |
| Old API client without `wrap=` | profile default |
| Frontend old version (no JS hybrid wrap) | server returns pre-wrapped `\n`-separated zh; frontend renders lines |

### Cost Runaway Protection

| Trigger | Action |
|---|---|
| Per-file cost > $0.10 (3× expected) | hard abort + 503 + log incident |
| Single file rewrite count > 100 segs | force `parallel_batches=4`; alert |
| Daily aggregate spend > $5 | rate limit new translations; admin notification |

## Cost & Runtime

Per 90-segment file:
- L1 K0: ~10s, ~$0.005
- L2 K2: ~10s, ~$0.005 (parallel with K0)
- L3 K4 rewrites: ~30s, ~$0.020
- L4 + L5: <1s
- **Total: ~50s, ~$0.030/file**
- vs current K0-only: ~10s, ~$0.005
- **Net delta: +40s latency, +$0.025/file**

With Mod M1 (cached K0 reuse) for existing files: ~$0.020/file delta.

## Open Issues + Mitigations (Section 5)

### G1: NAME_INDEX scalability
- **Risk**: cross-corpus F1=0% if entities not in hand-curated index
- **Mitigation**: Mod 1 auto-NER from EN capitalization + translit run heuristic; F1 metric becomes hybrid known+proxy

### G2: Glossary auto-feed sync
- **Risk**: glossary updates don't reach engine until restart
- **Mitigation**: Mod 2 glossary CRUD emits `glossary_updated` socket event; engine listener re-builds index

### G3: Pass 4 lock data loss
- **Risk**: silent corruption when all positions locked (rare but possible)
- **Mitigation**: Mod 3 escalates Pass 4 to explicit failure with red banner in proofread UI

### G4: F2 hallucination noise
- **Risk**: false positives reduce metric usefulness
- **Mitigation**: Mod 4 strict orthographic match (substring + key absence)

### G5: Frontend Node.js dependency burden
- **Risk**: adds new tooling/setup to vanilla JS frontend
- **Mitigation**: Mod 5 use Playwright headless instead of Node test runner

### G6: Migration UX
- **Risk**: existing files don't get A3 benefits without manual re-translate
- **Mitigation**: Mod 6 lazy on-render translation triggers background A3 if missing K2/K4

### G7: CPS gate enforcement
- **Risk**: design mentions CPS but pipeline doesn't enforce
- **Mitigation**: Mod 7 a3_ensemble layer computes CPS and gates winner selection

### G8: Production rate limit
- **Risk**: 3-way parallel × N users → OpenRouter 429
- **Mitigation**: Mod 8 translation queue + per-user semaphore + global semaphore=4 + job coalescing

### Important Mods (M1-M3)
- **M1**: Cached K0 reuse — saves cost on existing files
- **M2**: Canonical fixture regeneration script — keeps fixtures in sync as lock chain evolves
- **M3**: CI cost ceiling — prevents accidental spend ($10/month cap)

### Out of Scope (deferred)
- spaCy / proper NER (uses regex heuristic for now; spaCy can be added later if F1 plateau)
- Per-corpus auto NAME_INDEX seed extension from production usage (manual maintenance)
- F1 sensitivity analysis to NAME_INDEX coverage (operational concern)
- Visual regression testing (font metrics, kerning) — Phase 2 fontsdir already addresses TTF parity
