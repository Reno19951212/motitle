## Operational quirks (read before debugging anything auth-related)

### `admin_p3` password gets reset by pytest fixtures
Full backend pytest run rewrites `admin_p3` in `backend/data/app.db` via test fixtures. After any pytest run, restore with:
```bash
cd backend && source venv/bin/activate && set -a && source .env && set +a
python -c "from auth.users import update_password; update_password('data/app.db', 'admin_p3', 'AdminPass1!')"
```

⚠️ `auth.users.update_password(db_path, username, new_password)` takes **username** (string), not user id. Earlier handoff attempts wasted time on this.

### Correct AUTH DB path
`app.config['AUTH_DB_PATH']` defaults to `DATA_DIR / 'app.db'` — NOT `data/auth.db` (which is a stale leftover from older code). Always use `data/app.db`.

### `/login` rate limit (Flask-Limiter)
10 attempts per minute per IP. Tests that login per `beforeEach` will trigger 429 if ≥11 cases run consecutively against same backend. Workaround: split test runs by suite, or set `RATELIMIT_ENABLED=False` in `backend/.env` during local dev (it's already `False` in `conftest.py`).

### Backend boot requires `FLASK_SECRET_KEY` env (v3.13 hard requirement)
Always `set -a && source backend/.env && set +a` before `python app.py`. The `.env` file is gitignored and lives at `backend/.env`.
