## Root-cause grouping (where the fix should land)

### Group 1 ‚Äî Field-shape drift (8 of 13 bugs)

V6 stores translation data under `entry["translations"][i]["by_lang"][lang]["text"]` (+ `.status`). Legacy code reads from `entry["segments"]` or `t["zh_text"]` / `t["status"]`. A coordinated fix landing one of these:
- (a) `_persist_by_lang` backfills `t["zh_text"]` + `t["status"]` mirrors of `by_lang.zh.*`
- (b) Each legacy endpoint reads `by_lang[lang].text` (or `.status`) preferentially with `zh_text` fallback

‚Ä¶closes the following bugs at once:

| ID | Surface | Symptom |
|---|---|---|
| A-1 | `GET /api/files` per-row | `active_kind` missing ‚Üí Proofread defaults to Profile mode for V6 files |
| A-2 | `GET /api/files/<id>/translations` | `zh_text` empty (only `by_lang.zh.text` populated) |
| B-3 | `POST .../translations/approve-all` | filters on `t["status"]` ‚Üí matches 0 V6 rows |
| B-4 | `GET .../subtitle.{srt,vtt,txt}` | iterates `entry["segments"]` (empty for V6) ‚Üí all body empty |
| B-5 | `PATCH .../translations/<idx>` | writes only `zh_text`, leaves `by_lang.zh.text` stale ‚Üí **data integrity bug** (user edits silently lost on render) |
| B-6 | Render `subtitle_source=zh` | reads `zh_text` ‚Üí burns 81/82 silent segments |

Recommended approach (a) is simpler ‚Äî one helper in `_persist_by_lang` to mirror fields. Approach (b) is more correct but touches many sites.
