## Executive Summary
- **Verdict**: ✅ Merge — v3.17 zero regression

### Key Findings

1. **v3.17 itself zero regression**：preset trim、Whisper schema narrow、stub delete、profile migration 全部隔離通過。Video 1 ASR text 100% identical baseline ↔ post（zero drift）；MT latency 由 82.0s 改善至 65.7s（-20%）+ Video 2 由 75.5s 至 24.5s（-67%）。Glossary 違規由 8 改善至 5（Video 1）/ 0 → 1（Video 2，small uptick）。
2. **MT 質量正常**：Video 1 post 166 個 segments 入面只有 9 個（5.4%）空 zh_text；Video 2 85 個 segments 入面 2 個（2.4%）空。屬於 `alignment_pipeline.py` LLM-marker fallback 正常邊界 case，**唔係 bug**。
3. **「Task 11 post-snapshot 61% empty」係 transient state**：Task 11 期間發生過 backend swap（舊 PID 96344 仍 alive + 新 backend 啟動掛 FLASK_SECRET_KEY），post snapshot 捕捉到中間 broken state 嘅數據。Investigation phase（add debug log + re-trigger MT）證實正常運作下 empty rate < 6%。Investigation 後嘅 fresh re-snapshot 即係呢份報告嘅數據。
4. **`alignment_pipeline.py` glossary field compat fix 真係必要**：v3.15 multilingual refactor 漏咗呢個 file，當 `alignment_mode: "llm-markers"` + glossary 有 entries 時 KeyError silent fail。v3.17 加咗 backward-compat fallback (`source` → `en` field)。
5. **Video 2 baseline vs post 仍唔可直接比較**：Video 2 baseline 原本用 Cantonese-language profile 跑（ASR 中文），但 registry 冇記 profile_id，post snapshot 經 fallback 用 active profile（`asr.language=en`），ASR 變英文。**屬於 profile linkage data 問題，唔係 v3.17 regression。建議獨立任務 backfill registry profile_id**。
