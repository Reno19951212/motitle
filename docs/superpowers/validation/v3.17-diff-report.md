# v3.17 Validation Diff Report

## Executive Summary
- **Verdict**: ⚠️ Accept v3.17 with notes (pre-existing MT bug surfaced)
- **Date**: 2026-05-15
- **Videos tested**: 2

### Key Findings

1. **v3.17 itself is non-regressive**：preset trim、Whisper schema narrow、stub delete、profile migration 全部冇引入新問題。Video 1 ASR 166 segments identical baseline ↔ post（zero text drift）；MT latency 反而由 baseline 平均 78s 減到 49s（-37%）。
2. **Surfaced pre-existing alignment_pipeline.py bug**：Video 1 嘅 post MT 有 61.4% segment (102/166) 嘅 `zh_text` 係空字串。Pattern：前 82 個 segment 全部空，之後零星出現有內容嘅 segment。屬於 `alignment_pipeline.py` LLM-marker parsing 失敗，silent fallback 為空（spec 提及 `time_proportion_fallback` 行為）。**呢個 bug pre-date v3.17，但 v3.17 強制走呢個 code path（active profile 用 `alignment_mode: "llm-markers"`）**。
3. **Glossary field compat fix made during validation**：發現 [backend/translation/alignment_pipeline.py](backend/translation/alignment_pipeline.py) line 78/81 仍用 v3.14 嘅 `e['en']`/`e['zh']` 而唔係 v3.15 嘅 `source`/`target` — KeyError 令到所有 MT job silently 失敗。已加 backward-compat fallback 入呢個 branch。
4. **Video 2 baseline vs post 唔可直接比較**：Video 2 baseline 用 Cantonese-language profile 跑（ASR 中文），但 registry 入面冇記 `profile_id`，導致 post snapshot fallback 去 active profile (`asr.language=en`)，post ASR 出咗英文。**ASR text delta 嘅 65 changed + 39 dropped 主要係呢個 profile mismatch 而非 v3.17 影響**。

## Conclusion

**Recommendation: Merge v3.17 with concurrent fix for alignment_pipeline.py**

v3.17 嘅 4 個範圍（preset trim / Whisper schema / stub delete / profile migration）全部 isolated 唔造成 regression。Validation 過程意外發現兩個 pre-existing MT 問題：

| Issue | Origin | Status |
|---|---|---|
| `alignment_pipeline.py` glossary field 用 `en`/`zh`（v3.14 schema） | v3.15 multilingual refactor 漏 migrate 呢個 file | ✅ Fix 已喺 v3.17 branch 加 backward-compat |
| Marker-split silent fallback → 61% empty zh_text | Pre-existing，深層 alignment_pipeline.py 問題 | ❌ **未 fix** — 應該開獨立 v3.18 branch 跟進 |

**建議 merge path**：
1. Merge `chore/v3.15-cleanup-2026-05-13` 入 `dev`（含 v3.16 + v3.17）
2. 立即開 v3.18 branch 處理 marker-split bug
3. 喺 v3.18 release 之前，提醒用戶 `alignment_mode: "llm-markers"` 唔可靠，建議改用 `alignment_mode: ""`（batched）或 `alignment_mode: "sentence"` (sentence pipeline) — 喺 Profile UI 加警告

**Rollback option**（如果 v3.18 fix 時間長）：將 active profile 嘅 `alignment_mode` 改為 `""`，跳過 LLM-markers pipeline。已有 segments 嘅 ZH translation 保留。

### Specific Concerns + Suggested Follow-up

1. **v3.18 priority — alignment_pipeline.py marker-split debug**：reproduce 步驟已記錄（用 active profile + Video 1 即可），需要：
   - Read LLM raw response 對應第一個 batch 嘅 marker output
   - 確認 marker `[N]` placement 同 segment boundary 是否一致
   - 修 `parse_markers()` + `time_proportion_fallback()` 嘅 fallback edge case
2. **registry profile_id backfill**：v3.10 之前嘅文件冇記 profile_id；考慮一次性 migration 將 `profile_id=None` 嘅文件 backfill 做當前 active profile id，避免將來 re-run 用錯 profile
3. **Active profile language=en + Video 2 Cantonese audio**：呢個 setup 唔合理。考慮：(a) 加 multi-language profile auto-switch，或 (b) 喺 Profile UI 警告用戶 audio language 同 profile language mismatch
