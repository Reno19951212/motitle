## What this report measures

Whether v3.18 Stage 2's 3 default prompt rewrites (A) reduce MT formulaic phrase over-use without regressing other quality dimensions.

**The削減 hypothesis (from v3.17 MT quality research):**
- Old prompts contained 4 EN→ZH hardcoded mappings (`persistent → 傷病纏身` etc) and 3 connector examples (`在…方面`, `就此而言`, `儘管…但`)
- LLM in-context learning treated these as HARD mappings, causing 「傷病纏身」 to appear 15× / 「就此而言」 14× / 「儘管」 13× / 「真正」 24× across 166 segments
- New削減版 prompts: drop hardcoded mappings + add anti-formulaic rule

**Stage 2 success criteria** (from spec):
- Formulaic phrase frequencies drop by ≥60%
- Empty rate stays ≤6% (v3.17 was 5.4%)
- ZH/EN ratio distribution maintained (0.4-0.7 target)
- No new hallucination class introduced (manual spot-check on 5 known bad segments)

### Prerequisites
1. Backend running on `http://localhost:5001`

### Steps

```bash
# 1. Switch to v3.18 branch (already on it if you're reading this in repo)
git checkout chore/v3.18-stage2-prompt-override

# 2. Restart backend to load the削減版 default prompts
pkill -f "python app.py"
cd backend && source venv/bin/activate
nohup python app.py > /tmp/whisper-backend.log 2>&1 &
sleep 3 && curl -sf http://localhost:5001/api/health

# 3. Identify the Video 1 file id (find from the existing baseline JSON filename)
VIDEO1_ID=abdbe1873542   # change if your baseline file id differs

# 4. Clear any per-file prompt_overrides on this video (to test the削減版 default, not any file-level override)
curl -X PATCH http://localhost:5001/api/files/$VIDEO1_ID \
  -H "Content-Type: application/json" \
  -d '{"prompt_overrides": null}'

# 5. Trigger re-translate (this takes 5-10 min)
curl -X POST http://localhost:5001/api/translate \
  -H "Content-Type: application/json" \
  -d "{\"file_id\":\"$VIDEO1_ID\"}"
# Wait for translation_status=done — monitor via:
#   curl -sf http://localhost:5001/api/files/$VIDEO1_ID/translations/status
# OR watch the dashboard UI

# 6. Capture post-snapshot
cd backend
python scripts/v317_validation.py snapshot \
    --file $VIDEO1_ID \
    --output ../docs/superpowers/validation/v3.18-stage2-post.json

# 7. Generate diff report
python scripts/v317_validation.py diff \
    --baseline ../docs/superpowers/validation/v3.17-baseline-$VIDEO1_ID.json \
    --post ../docs/superpowers/validation/v3.18-stage2-post.json \
    --output ../docs/superpowers/validation/v3.18-stage2-numerical-diff.md
```
