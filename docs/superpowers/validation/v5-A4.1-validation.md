## Architecture Note

The dual-ASR Verifier was *designed* to catch these via LLM-as-judge, but in practice the granularity mismatch between mlx-whisper (90 short segs) and qwen3-asr (13 mega-chunks) means the verifier silently defaults to "keep primary" on 91% of segments. This quick fix bypasses the verifier value-add for these specific Whisper artifacts; the deeper verifier re-segmentation work remains tracked for Phase 5+.

Filter order is semantically critical (verified in T2 code-review): dedupe MUST run before tail filter, otherwise a cascade-of-orphans case (3× `"vowels"` repeats at end of clip) would fail to trigger the tail filter's gap condition.
